# Step 149 - 内测反馈记录模板与分级规则

## 本轮目标

1. 为 owner-controlled 内测补齐反馈记录模板。
2. 明确阻塞 / 主要 / 次要 / 建议四类优先级。
3. 明确哪些反馈进入修复，哪些进入后续规划。
4. 不改业务代码、不改 bridge、不改鉴权、不改接口、不改路由。

## 为什么做这个

Step 148 已明确谁可以参与内测、账号如何发放、哪些链路可以测、哪些能力不要测。

但如果没有反馈模板，真实内测会出现：

1. 反馈缺少页面、步骤、预期和实际结果。
2. 视觉建议、功能建议和阻塞 bug 混在一起。
3. 开发轮次被“感觉问题”带跑偏。

因此 Step 149 先把反馈分级和进入修复的条件写清楚。

## 当前修复入口边界

只有满足以下条件才进入修复：

1. 有明确页面或接口。
2. 有复现步骤。
3. 有预期结果和实际结果。
4. 严重程度为阻塞或主要，或次要问题已成组出现。
5. 修复范围不违反当前冻结线。

当前冻结线继续保持：

1. bridge 主线冻结。
2. 展示 polish 线维护态。
3. 媒体线收住。
4. 旧外卖模块不直接删除。
