# Step 102 - 校园兼职视觉体系与 admin 外壳刷新

## 本轮目标

在 Step 101 已从根因修复 admin 中文乱码后，本轮进一步处理用户截图中“仍像苍穹外卖后台”的视觉问题。目标是先从高曝光 admin 入口建立校园兼职平台的基础视觉方向，而不是继续改业务能力。

## 已完成项

1. 刷新全局主题变量：
   - 将主色从旧外卖橙红调整为校园 teal / deep-blue / lime 方向。
   - 调整基础字体栈，优先使用更适合中文后台的 `HarmonyOS Sans SC / MiSans / Segoe UI`。
   - 更新主阴影色与强调色，减少旧外卖风格痕迹。
2. 刷新 admin 主框架 `MainLayout.vue`：
   - 顶部品牌从“智慧餐饮 / 管理系统”改为“校内兼职 / Campus Ops”。
   - 菜单按“校园运营”和“旧模块兼容”分组。
   - 旧外卖模块菜单改为兼容口径，例如“分类兼容 / 商品兼容 / 套餐兼容 / 订单兼容”。
   - 侧边栏、顶部导航、面包屑、用户角色展示统一为校园兼职运营风格。
3. 刷新 admin 仪表盘 `Dashboard.vue`：
   - 首页标题从旧 dashboard 文案改为 `Campus Part-time Ops` 与“校内兼职平台运营总览”。
   - 卡片指标改为“模拟流水 / 订单快照 / 用户规模 / 平均单值”。
   - 图表说明明确仍复用旧统计接口，但展示口径改为试运营运营视角。
   - 卡片、图表和最近订单区改为校园运营视觉层级。
4. 刷新运营人员页 `Employee.vue`：
   - 页面从“员工管理”改为“运营人员”。
   - 增加顶部校园运营人员说明区、筛选说明卡片和人员列表标题。
   - 优化空态、状态文字、操作区、表格层级和分页布局。
   - 保留原有员工增删改查、状态切换、分页和搜索行为。

## 明确未改

- 未改 bridge。
- 未改 `frontend/src/utils/request.js`。
- 未改任何 `campus-*` API 文件运行时行为。
- 未改 token 附着逻辑。
- 未改 API 调用顺序。
- 未改路由。
- 未改后端代码、数据库、订单 / 售后 / settlement / 异常业务语义。
- 未新增页面。
- 未删除旧外卖模块；只把旧模块在 admin 菜单中标记为兼容模块。

## 遗留问题

- 本轮只处理 admin 主框架、仪表盘和运营人员页，旧外卖兼容模块内部页面仍可能保留旧视觉。
- customer / courier 页面此前已完成局部 polish，但尚未和本轮 admin 外壳建立完整统一设计系统。
- 如果浏览器仍显示旧样式，优先强制刷新或清理 Vite / 浏览器缓存。

## 下一步建议

1. 用户本地先复核 admin 三个入口是否已经不再出现旧外卖视觉和中文乱码。
2. 若视觉方向确认，再做下一轮小范围统一：优先检查登录页和旧模块兼容页的视觉割裂。
3. 不建议一次性重写全部页面；继续采用“高曝光入口优先，小步验证”的方式。
