# Step 163 - 内测分发材料收口

## 本轮目标

1. 基于 Step 162 已同步 GitHub 与内测服务器的状态，补齐小范围内测分发材料。
2. 不继续开发新功能。
3. 不改业务代码、不改 bridge、不改鉴权、不改接口、不改路由、不新增页面。
4. 形成一份测试者能看懂、owner 能执行的内测分发总入口。

## 为什么本轮做这个

当前项目已经具备：

1. GitHub 主线已更新。
2. 内测服务器已同步并通过远端 smoke。
3. Android 用户端和兼职端能构建、安装和启动。
4. 真机公网主链路曾跑通到 `COMPLETED`。

下一步如果要让别人试用，最容易翻车的不是“缺一个新功能”，而是：

1. 不知道装哪个包。
2. 不知道用哪个账号。
3. 不知道哪些能力是模拟的。
4. 不知道怎么反馈问题。
5. 把内测包误解成正式上线版本。

所以本轮先补分发材料，而不是继续写代码。

## 当前判断

当前已经具备“给少量可信测试者发包”的基础材料：

1. 有 APK 构建脚本说明。
2. 有安装说明。
3. 有测试账号发放边界。
4. 有用户端 / 兼职端 / 管理后台测试链路。
5. 有已知限制说明。
6. 有反馈模板。
7. 有 owner 发包前检查清单。

这仍然不等于公开公测。公开公测前仍需：

1. release 签名包。
2. HTTPS / 域名 / 证书。
3. 更完整的隐私和安全说明。
4. 多账号、弱网、后台切回和重复下单回归。
5. 更稳定的问题收集与版本发布流程。
