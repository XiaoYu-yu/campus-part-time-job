# 内测分发包说明

## 目标

本说明用于把当前项目整理成可发给少量可信测试者的 owner-controlled 内测包。

当前定位：

```text
本地 / 内测型试运营可用
不等于公开公测
不等于正式产品上线
```

## 适用范围

适合：

1. owner 本人。
2. 1 到 3 名可信同学或朋友。
3. 用于验证登录、发单、接单、确认、后台查看等主链路。
4. 用于收集明显阻塞、主要问题和可用性反馈。

不适合：

1. 公开放给陌生用户。
2. 承诺真实履约。
3. 承诺真实支付、退款、打款。
4. 承诺正式 SLA。
5. 承诺真实地图导航、真实定位轨迹或正式调度。

## 分发包内容

每次准备发给测试者前，至少准备下面 5 类材料：

1. Android 用户端 APK。
2. Android 兼职端 APK。
3. 测试账号说明。
4. 已知限制说明。
5. 反馈模板。

推荐本地归档结构：

```text
internal-trial-package/
  user-app.apk
  parttime-app.apk
  README-安装说明.md
  README-测试账号.md
  README-已知限制.md
  feedback-template.md
```

该目录用于私下分发，不建议直接提交到仓库。

## APK 生成

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\trial-operation\build-android-qa-apks.ps1 -Mode public
```

输出目录：

```text
project-logs/campus-relay/runtime/android-qa-apks/<timestamp>/
```

注意：

1. 该目录已被 git 忽略。
2. APK 二进制文件不要提交到公开仓库。
3. Debug APK 只适合 owner-controlled 内测，不等于正式 release 包。
4. 若给更多人安装，下一阶段应补 release 签名包。

## Android 安装说明

测试者安装前：

1. 确认手机系统允许安装未知来源应用。
2. 先安装 `user-app.apk`。
3. 再安装 `parttime-app.apk`。
4. 如果旧版本无法覆盖安装，先卸载旧版本再安装。
5. 安装后桌面应看到两个应用：
   - `用户端`
   - `兼职端`

如果打开后一直白屏或无法登录：

1. 先确认手机网络可访问内测服务器。
2. 关闭 App 后重开。
3. 仍失败则截图并按反馈模板提交。

## 测试账号说明

测试账号由 owner 私下发放，不写入公开仓库。

建议发放格式：

```text
用户端账号：
手机号：
密码：

兼职端账号：
手机号：
密码：

管理后台账号：
手机号：
密码：
管理端访问地址：由 owner 私下提供
```

账号发放原则：

1. 每个测试者只拿自己需要的角色账号。
2. 不把账号密码发到公开群、公开 README 或截图里。
3. 测试结束后可重置密码或重置测试数据。
4. 若反馈截图包含手机号、姓名、地址等信息，提交前应打码。

## 建议测试链路

### 用户端

建议按顺序测试：

1. 打开 `用户端`。
2. 登录。
3. 进入首页。
4. 发布校园代送单。
5. 完成模拟支付。
6. 查看订单进度。
7. 等兼职端送达后，进入订单结果页。
8. 点击确认收到。
9. 确认订单显示已完成。
10. 如需要，查看售后结果页。

不要测试：

1. 真实支付。
2. 真实退款。
3. 真实打款。
4. 输入真实隐私地址。
5. 高频刷单或压测。

### 兼职端

建议按顺序测试：

1. 打开 `兼职端`。
2. 登录。
3. 查看工作台。
4. 查看可接任务。
5. 接单。
6. 确认取餐。
7. 开始配送。
8. 确认送达。
9. 如遇异常，尝试上报异常。
10. 回看已完成订单。

不要测试：

1. 真实地图导航。
2. 真实定位轨迹。
3. 真实骑手调度。
4. 恶意重复点击或压测。

### 管理后台

建议 owner 或熟悉项目的人测试：

1. 登录后台。
2. 查看运营总览。
3. 查看订单相关后台页。
4. 查看兼职运营页。
5. 查看异常历史和处理状态。
6. 查看售后执行页。
7. 查看结算只读运营页。

不要测试：

1. 删除旧外卖兼容模块。
2. 改 bridge。
3. 改鉴权。
4. 改 token。
5. 改真实资金链路。

## 已知限制

当前内测版本仍有这些限制：

1. 没有真实支付，只有模拟支付。
2. 没有真实退款。
3. 没有真实打款。
4. 没有正式 HTTPS / 域名 / 证书收口。
5. Android 当前仍是内测 Debug / QA 包，不是应用商店 release 包。
6. 腾讯地图能力只作为内测辅助能力，不承诺正式导航。
7. 旧外卖模块仍作为兼容模块保留，不直接删除。
8. bridge 主线处于 `Phase A no-op` 冻结态，不在内测期间随意收口。
9. 不承诺并发能力或压力测试结果。
10. 不承诺正式隐私合规、备案、短信、推送等生产能力。

## 反馈模板

请让测试者按下面格式反馈：

```text
反馈人：
测试设备：
测试角色：用户端 / 兼职端 / 管理后台
页面入口：
操作步骤：
预期结果：
实际结果：
是否可复现：是 / 否 / 未确认
截图或录屏：
严重程度：阻塞 / 主要 / 次要 / 建议
备注：
```

严重程度说明：

1. 阻塞：无法登录、打不开 App、无法继续主链路。
2. 主要：发单、支付、接单、取餐、送达、确认等核心动作失败。
3. 次要：页面错位、文案不清楚、空态不好看，但不影响继续测试。
4. 建议：希望新增功能或优化体验。

## owner 每次分发前检查

发包前先确认：

1. `origin/main` 是最新提交。
2. 内测服务器已同步最新代码。
3. `remote-smoke.ps1` 通过。
4. 用户端 APK 和兼职端 APK 均为同一轮构建产物。
5. 手机上能安装并打开两个 App。
6. 用户端能登录。
7. 兼职端能登录。
8. 管理后台能登录。
9. 测试账号已私下发放。
10. 已向测试者说明：当前不是正式上线版本。
