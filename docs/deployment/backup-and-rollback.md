# 备份与回滚说明

## 数据库备份

上线前必须先做全量备份。

### MySQL 逻辑备份示例

```bash
mysqldump -h %DB_HOST% -P %DB_PORT% -u %DB_USERNAME% -p %DB_NAME% > backup_full.sql
```

### 建议策略

- 每次上线前做一次全量备份
- 高风险变更前额外做一次即时备份
- 保留最近多份备份，并记录生成时间和对应版本

## 应用回滚

### 后端回滚

1. 保留至少一个上一个稳定版本的 jar 包
2. 如果新版本异常，停止当前进程
3. 启动上一个稳定版本
4. 验证登录、下单、订单查询、统计接口

### 前端回滚

1. 保留上一个稳定版 `dist/`
2. 如果新版本异常，直接回滚静态资源
3. 清理 CDN 或浏览器缓存

## 数据库回滚

### 回滚原则

- 优先执行明确的回滚 SQL
- 若没有安全的增量回滚脚本，则恢复到上线前全量备份

### 推荐做法

- 每个迁移脚本在发布前评估是否需要对应回滚脚本
- 高风险 DDL 变更单独安排维护窗口
- 对删除字段、修改字段类型等不可逆变更，先走双写或灰度策略
