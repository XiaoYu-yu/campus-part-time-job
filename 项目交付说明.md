# 项目交付说明

## 交付目标

当前交付版本以“你这台 Windows 电脑可直接打开和运行”为目标整理，默认开发数据库已经切到本机 `MariaDB`，前端使用 `HBuilder X` 打开，后端使用 `IntelliJ IDEA` 打开。

## 当前项目包含什么

- 管理后台和用户端两套业务页面
- 真实前后端接口联调
- MariaDB 本地开发配置
- 后端 `Maven Wrapper`
- 完整的修复日志、生产准备日志和部署文档

## 这次整理删除了什么

- 前端构建产物 `frontend/dist`
- 前端依赖目录 `frontend/node_modules`
- 后端编译产物 `backend/target`
- 临时运行日志 `project-logs/runtime`
- 私有编辑器配置 `.vscode`、`frontend/.vscode`
- 私有 AI 目录 `.trae`
- 历史噪音文档 `frontend/美化升级修改说明.md`

这些文件都不影响项目交付和再次启动，需要时会重新生成。

## 为什么改成 Maven Wrapper

- 公开仓库更标准
- 克隆仓库后不需要单独准备 Maven 发行包
- IDEA 和命令行都能直接使用 `backend/mvnw.cmd`
- 你本机原来的 `apache-maven-3.9.14` 仍可保留在目录里备用，但不再纳入 Git 版本控制

## 为什么改用 MariaDB

- 你已经装好了 `MariaDB`
- 你需要中文数据库管理界面，可以配合 `HeidiSQL` 或 `DBeaver`
- 这个项目原本就更接近 MySQL/MariaDB 语法体系
- 比继续用 H2 更接近真实数据库环境

## 如何用 IDEA 启动后端

### 1. 打开项目

- 用 `IntelliJ IDEA` 打开目录：[backend](D:/20278/code/show_shop1/backend)

### 2. 检查 JDK

- Project SDK 设为 `JDK 17`

### 3. 检查 Maven

- 优先选择项目自带的 `Maven Wrapper`
- 后端目录中已有：
  - `mvnw.cmd`
  - `.mvn/wrapper/`

### 4. 启动配置

- 主类：
  - `com.cangqiong.takeaway.TakeawayApplication`
- Program arguments：

```text
--spring.profiles.active=dev
```

### 5. 启动成功后

- 后端地址：[http://localhost:8080](http://localhost:8080)
- 店铺状态接口：[http://localhost:8080/api/public/shop/status](http://localhost:8080/api/public/shop/status)

## 如何用 HBuilder X 启动前端

### 1. 打开目录

- 用 `HBuilder X` 打开目录：[frontend](D:/20278/code/show_shop1/frontend)

### 2. 说明

- 这个项目是标准 `Vue 3 + Vite` 项目
- `HBuilder X` 这里只作为编辑器使用
- 真正启动前端还是靠终端命令，不是 HBuilder 自带的 uni-app 运行按钮

### 3. 在终端执行

```bash
cd D:\20278\code\show_shop1\frontend
npm install
npm run dev
```

### 4. 启动成功后

- 前端地址：[http://localhost:5173](http://localhost:5173)

## 常见问题

### 1. 后端启动失败

先检查：

- 是否使用了 `JDK 17`
- 是否传了 `--spring.profiles.active=dev`
- MariaDB 服务是否已经启动

### 2. 前端打开后接口报错

先检查：

- 后端是否已经启动
- 前端是否通过 `npm run dev` 启动
- 后端端口是否仍是 `8080`

### 3. 想看数据库表和数据

直接用 `HeidiSQL` 或 `DBeaver` 连接本机 `127.0.0.1:3306`。
